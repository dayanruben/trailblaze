Usage: trailblaze waypoint validate [-hV] --def=<defId> [--root=<root>]
                                    [--session=<session>] [--step=<step>]
                                    [<positionalLogFile>]
Validate that a specific waypoint definition matches a captured screen state.
      [<positionalLogFile>] Path to *_TrailblazeLlmRequestLog.json (required
                              unless --session/--step given)
      --def=<defId>         Waypoint id to validate (required)
  -h, --help                Show this help message and exit.
      --root=<root>         Additional directory to scan for *.waypoint.yaml
                              files (default: ./trails, resolved against the
                              current working directory). Pack waypoints are
                              always included regardless of --root.
      --session=<session>   Session log directory (containing
                              *_TrailblazeLlmRequestLog.json files)
      --step=<step>         1-based index of the step within the session
                              (default: last step)
  -V, --version             Print version information and exit.
