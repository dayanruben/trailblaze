Usage: trailblaze migrate-trails [-hV] [--fail-on-dropped-content]
                                 [-o=<outputPath>] <input-dir>
Migrate a directory of legacy *.trail.yaml files (plus optional blaze.yaml)
into a single unified trail.yaml file. Does NOT delete the input files. Hidden
from `--help` because almost no one has v1 trails — this is a one-shot
migration utility, not part of the public surface. Still callable via
`trailblaze migrate-trails <input-dir>` when needed.
      <input-dir>   Directory containing one or more `<classifier>.trail.yaml`
                      files. The filename minus `.trail.yaml` is the device
                      classifier (e.g. `android-phone.trail.yaml` →
                      `android-phone`).
      --fail-on-dropped-content
                    Exit non-zero (ASSERTION_FAILED) when the migration drops
                      input content that did not round-trip — schema-unknown
                      keys, or sibling keys in a tool entry the tool decoder
                      discards. Off by default: a lossy migration still writes
                      a usable file with leading DROPPED warning comments and
                      exits 0. Turn this on in a pipeline that must refuse to
                      chain (e.g. `migrate-trails && commit`) on a lossy
                      migration.
  -h, --help        Show this help message and exit.
  -o, --output=<outputPath>
                    Path to write the unified file. Defaults to
                      `<input-dir>/trail.yaml`.
  -V, --version     Print version information and exit.
