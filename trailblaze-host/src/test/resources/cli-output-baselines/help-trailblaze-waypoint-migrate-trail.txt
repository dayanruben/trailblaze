Usage: trailblaze waypoint migrate-trail [-hV] [--write] --session=<sessionDir>
       <trailFile>
Mechanically migrate legacy `selector` (Maestro-shape) → `nodeSelector`
(accessibility
shape) for every selector-bearing tool in a trail YAML, using the captured
session
logs to deterministically resolve each selector through the same matcher the
runtime
uses for taps.
Defaults to dry-run (unified diff on stdout). Use `--write` to apply the
migration in
place. Pair with a recorded session log directory (`--session`) for the same
trail.
      <trailFile>   Path to the trail YAML file to migrate.
  -h, --help        Show this help message and exit.
      --session=<sessionDir>
                    Session log directory containing *_TrailblazeLlmRequestLog.
                      json files captured
                    during a recorded run of this trail. The Nth
                      selector-bearing tool in the YAML
                    is paired with the Nth log step that carries both
                      viewHierarchy and
                    trailblazeNodeTree.
  -V, --version     Print version information and exit.
      --write       Overwrite the trail file in place with the migrated YAML.
                      Default is dry-run:
                    print a unified diff for review without changing the file.
