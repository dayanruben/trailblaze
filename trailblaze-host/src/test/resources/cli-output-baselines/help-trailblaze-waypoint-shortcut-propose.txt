Usage: trailblaze waypoint shortcut propose [-hV] [--idempotence-check]
       [--fingerprint-agreement=<ratio>] [--min-support=<n>] [--out-dir=<dir>]
       [--root=<path>] --sessions=<dir> --target=<id> [--top-k=<n>]
Analyze a session set for (A->B) transitions and emit draft shortcut YAMLs.
Each surviving proposal lands as a JSON+YAML pair under --out-dir, ready for
the pipeline's verify + auto-PR steps.
      --fingerprint-agreement=<ratio>
                            Fraction of supporting sessions that must share the
                              dominant action fingerprint
                            (default: 0.67). Sessions disagreeing on the
                              procedure short-circuit the proposal.
  -h, --help                Show this help message and exit.
      --idempotence-check   Re-run after applying the surviving proposals
                              in-memory; fail (exit 1) if any new proposal
                              surfaces.
      --min-support=<n>     Minimum distinct sessions for a transition to be
                              proposed. Default: 3.
      --out-dir=<dir>       Output directory for proposal sidecars. Default: ./.
                              waypoints_shortcut/proposals/. Wiped at the start
                              of each run.
      --root=<path>         Override the waypoint-root dir for waypoint +
                              existing-shortcut discovery. (Convention: .
                              /trails)
      --sessions=<dir>      Directory containing one or more session
                              subdirectories with *_AgentDriverLog.json files.
                              Each subdirectory is one session.
      --target=<id>         Trailmap id. Resolves --root to
                              <workspace>/trailmaps/<id>/waypoints/ and
                              supplies the trailmap's app_ids for templated
                              selector expansion.
      --top-k=<n>           Process at most the top-K surviving proposals by
                              hit count. The replay step
                            is expensive enough that v1 caps weekly throughput
                              here. Default: 5.
  -V, --version             Print version information and exit.
